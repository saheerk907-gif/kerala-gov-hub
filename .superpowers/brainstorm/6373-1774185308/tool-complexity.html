How interactive should the new PDF tools be?

Pick the complexity level that fits your users best

A

Simple & Fast

Upload → click button → download. No previews, no drag-and-drop ordering. Works for users who just need a quick result.

  • Merger: upload files → merge → download
  • Splitter: upload + type page range → split → download
  • PDF to Text: upload → extract → copy/download text
B

Interactive with Previews

Drag-to-reorder file list for merger, visual page thumbnails for splitter, paginated text view for extractor.

  • Merger: drag files to reorder before merging
  • Splitter: click thumbnail pages to select them
  • PDF to Text: see text per page with copy buttons
C

Mixed — Simple for now, build on it later

Start with Simple & Fast for all three, but design it cleanly enough to add interactivity in a future update.

  • No drag-and-drop yet, but clean file list UI
  • Text range input for splitter (e.g. "1-3, 5, 7-9")
  • Full text output with single copy button
